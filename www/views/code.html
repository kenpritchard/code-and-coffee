Spring and Google App Engine

For the 2013 Orlando Google I/O Extended event I did a presentation about converting a simple Spring project to run on Google App Engine. The concept was to take a small example Spring project and demonstrate what would be necessary to port it to GAE. In addition, the final version takes advantage of Spring dependency injection to easily switch to using Datastore for persistence.

A primary concern when deploying a Spring project on GAE is performance. Much of the power of Spring comes from having the Spring container create and inject all the dependencies when the application loads. However, GAE needs the application to be responsive so it is necessary to ensure that Spring either completes the initial wiring quickly or defers some of it. More information is available in the slides that were presented.

The projects are available on GitHub separately as the initial Spring project and the converted GAE project.
